## Blocking / Important Gaps

### G24-01
- **Severity**: Important
- **Design Requirement**: AGENTS requires all `except` blocks to log `category="error"` with `exception_type`, `traceback`, `context`, and `component` (`AGENTS.md:103`). The design spec defines `error` logs for any exception with those same fields (`docs/superpowers/specs/2026-07-02-myagentcli-design.md:985`).
- **Actual Implementation**: Broad `except Exception` coverage is much better, but global compliance is still incomplete. The new static guard only checks broad handlers (`tests/logging/test_exception_logging_static.py:36`, `tests/logging/test_exception_logging_static.py:107`) and log calls that already use `logger.exception` or `exc_info=True` (`tests/logging/test_exception_logging_static.py:84`, `tests/logging/test_exception_logging_static.py:119`, `tests/logging/test_exception_logging_static.py:125`). It does not require every `ExceptHandler` to either emit a structured error log or be explicitly documented/whitelisted as control flow.
- **Evidence**:
  - Unlogged YAML parse fallbacks remain: config AGENT frontmatter returns `{}` (`myagent/config/loader.py:369`, `myagent/config/loader.py:370`), skill frontmatter returns `None` (`myagent/skills/loader.py:62`, `myagent/skills/loader.py:63`), and memory frontmatter returns `{}` (`myagent/memory/store.py:160`, `myagent/memory/store.py:161`).
  - Unlogged optional/fallback imports remain: CLI list-session rich fallback (`myagent/cli/main.py:577`, `myagent/cli/main.py:578`), status-bar rich fallback (`myagent/cli/status.py:53`, `myagent/cli/status.py:54`), REPL syntax/fallback imports (`myagent/cli/repl.py:311`, `myagent/cli/repl.py:366`), memory semantic recall optional import logs `category="system"` rather than `error` (`myagent/memory/recall.py:43`, `myagent/memory/recall.py:45`), and context-window discovery silently passes on `ImportError` (`myagent/agent/engine.py:1129`, `myagent/agent/engine.py:1131`).
  - Unlogged parse/control exceptions remain in LLM/MCP paths: incomplete streamed tool-call args continue on `JSONDecodeError` (`myagent/llm/provider.py:651`, `myagent/llm/provider.py:653`), non-streaming malformed tool-call args become `{}` (`myagent/llm/provider.py:707`, `myagent/llm/provider.py:708`), MCP shutdown/receive/SSE timeout and cancellation paths return/pass/continue without error metadata (`myagent/tools/mcp/client.py:106`, `myagent/tools/mcp/client.py:154`, `myagent/tools/mcp/client.py:276`, `myagent/tools/mcp/client.py:392`, `myagent/tools/mcp/client.py:406`, `myagent/tools/mcp/client.py:790`).
  - Project detection and logging cleanup still silently degrade on filesystem/process exceptions: Git branch/status/version/structure fallbacks (`myagent/agent/project.py:173`, `myagent/agent/project.py:194`, `myagent/agent/project.py:235`, `myagent/agent/project.py:291`) and log handler/cleanup best-effort failures (`myagent/logging/logger.py:92`, `myagent/logging/logger.py:94`, `myagent/logging/logger.py:305`).
- **Impact**: Operators still cannot rely on `category="error"` queries to enumerate all caught exceptions. Traceback/context/component fields are available for improved broad handlers, but narrow parse, optional dependency, timeout, cancellation, filesystem, and fallback exceptions remain invisible or logged under non-error categories. This keeps the implementation short of both the repository convention and design logging contract.
- **Suggested Fix**: Decide whether the strict rule literally applies to every caught exception. If yes, update all remaining narrow handlers to use `logger.exception` or `logger.warning/error(..., exc_info=True, extra={"category": "error", "component": "...", "context": "..."})`, including expected fallback paths. If no, update AGENTS/design with explicit exemption categories such as optional dependency absence, normal cancellation, timeout polling, and expected parse-in-progress; then strengthen `tests/logging/test_exception_logging_static.py` to require every `ExceptHandler` to either have a structured error log or match a narrow, documented whitelist with a reason.
