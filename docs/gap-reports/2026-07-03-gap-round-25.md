## Blocking / Important Gaps

- **Severity**: Important
- **Design Requirement**: AGENTS requires every `except` block to log `category="error"` with `exception_type`, `traceback`, `context`, and `component` where component is documented as `llm/tool/agent/mcp` (`AGENTS.md:103`). The design spec gives the same `error` category fields and component values (`docs/superpowers/specs/2026-07-02-myagentcli-design.md:985`).

- **Evidence**:

- **Impact**: The implementation is structurally logging errors, but the schema no longer fully matches the documented contract. Add a small formatter/static test that fails if implementation and documented component enum drift again.

## Minor Gaps / Documentation Gaps

### G25-02
- **Severity**: Minor / Operational Risk
- **Design Requirement**: The design requires useful structured operational logs and also includes logging performance considerations such as async queueing and controlled log volume (`docs/superpowers/specs/2026-07-02-myagentcli-design.md:985`). AGENTS currently requires every `except` block to produce error metadata (`AGENTS.md:103`).
- **Actual Implementation**: The literal all-except rule is now enforced by turning several expected control-flow or fallback paths into `ERROR` traceback logs. Examples include normal queue drain races (`myagent/tools/mcp/client.py:350`, `myagent/subagent/pool.py:351`), expected SSE stream termination (`myagent/tools/mcp/client.py:442`), cancellation paths (`myagent/tools/mcp/client.py:170`, `myagent/tools/mcp/client.py:462`, `myagent/tools/mcp/client.py:854`, `myagent/cli/repl.py:536`), idle/exit input control flow (`myagent/cli/repl.py:341`, `myagent/cli/repl.py:373`, `myagent/cli/repl.py:626`), and optional/fallback imports (`myagent/agent/engine.py:1145`, `myagent/cli/repl.py:319`, `myagent/cli/main.py:578`, `myagent/cli/status.py:56`).
- **Impact**: This is not a direct G24 regression because the fix mostly adds logging to already caught paths. However, operators may see expected shutdown, prompt EOF, queue-drain races, optional dependency absence, and polling timeouts as ERROR records with full tracebacks. That can reduce the signal of real failures and increase JSONL volume, especially in long-running MCP/SSE sessions.
- **Suggested Fix**: If the project keeps the literal all-except requirement, add an explicit structured field such as `expected=true` / `control_flow=true` for benign exceptions and document how operators should filter them. If the intended behavior is lower-noise logging, revise AGENTS/design with narrow exemption categories and update the static guard to require either structured error logging or an explicit documented exemption.
