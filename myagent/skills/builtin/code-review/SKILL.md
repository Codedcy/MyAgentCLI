name: code-review
description: 代码审查，检查正确性、安全性、性能和可维护性问题
---

## 何时使用
用户要求 review、审查代码、检查 PR 时触发。

## 流程
1. 获取变更文件列表
2. 逐文件审查：安全 → 正确性 → 性能 → 可维护性
3. 输出结构化审查报告

## 审查维度
- **安全性**: SQL注入、XSS、敏感信息泄露、权限绕过
- **正确性**: 边界条件、空值处理、异常处理、并发安全
- **性能**: N+1查询、不必要的循环、内存泄漏、大对象复制
- **可维护性**: 命名清晰度、函数长度、模块耦合、文档完整性

## 注意事项
- 使用最高思考模式 (Think Max) 进行深度审查
